# Prompt 02 — Public marketing pages + pricing presentation + access states

## Objective
Build the public-facing pages that help sell the product and guide users into the app:
- landing page
- pricing page
- signup/login polish
- billing-required and access-related states

This prompt is about trust and conversion, not feature depth.

## Constraints
- Keep messaging clear and grounded.
- Make the UI visually appealing enough to support sales.
- Do NOT wire real Stripe yet.
- Do NOT use fake marketing fluff that the product cannot support.
- Keep copy editable and organized.

## Product message
The public promise at launch is:
- organize crew
- organize work
- track time
- review payroll
- keep access simple

The product is operations-first, not yet the full ranch suite.

## Tasks
1) Build the landing page.
Include:
- hero section
- value proposition
- feature overview for launch capabilities
- product screenshots/placeholder panels if real screenshots are not ready yet
- trust-building section
- clear calls to action

2) Build the pricing page.
Include:
- simple launch pricing presentation
- plan summary card(s)
- what is included at launch
- what is not included yet, if needed for honesty
- CTA into signup/checkout path placeholder

3) Polish login and signup pages.
- consistent styling with the rest of the brand
- helpful copy
- clear validation
- no ugly bare forms

4) Build access-state pages/components.
At minimum:
- billing required
- no ranch access
- insufficient role/access
- onboarding incomplete

5) Add a public/app visual bridge.
The jump from marketing pages into the app should feel like the same product.

6) Make these screens responsive and demo-ready.

## Acceptance Criteria
- Landing page feels intentionally designed and credible.
- Pricing page is clean and understandable.
- Login/signup feel polished, not placeholder-grade.
- Access states are humane and branded.
- A first-time visitor can understand the product quickly.

## Output Format
- Files changed
- Commands to run
- How to verify (click-by-click)
